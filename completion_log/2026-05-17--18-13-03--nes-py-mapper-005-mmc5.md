# nes-py mapper 005 MMC5

## Summary

Implemented a first-pass MMC5 mapper in `nes-py` with mapper 5 factory/registry support, PRG banking modes 0-3, CHR banking modes 0-3, banked/protected PRG RAM, ExRAM CPU access, mapper-owned nametable/fill mapping, extended-attribute background CHR selection, scanline IRQ status, multiplier registers, and backup/restore preservation through mapper cloning.

Added native mapper coverage in `nes_emu/test/nes_emu/mappers/test_mapper_MMC5.cpp` and public Python mapper coverage in `nes_py/tests/mappers/test_mapper_005_mmc5.py`, including the local Castlevania III fixture path.

## Secondary Feature Gaps

MMC5 vertical split registers `$5200-$5202` are preserved in mapper state but vertical split rendering is intentionally unsupported in this first pass. MMC5 pulse/PCM audio registers `$5000-$5015` are accepted as no-op writes and are intentionally unsupported. These gaps are documented in the mapper implementation and did not block the representative Castlevania III smoke path.
